# 第4课：文字如何变成机器能懂的数字？

## 前一节课内容回顾

**3 分钟回顾：从“神经网络”到“语言输入”**

上一节课我们学习了神经网络的基本结构：输入层接收信息，隐藏层加工信息，输出层给出结果；参数可以理解为不同线索的重要程度，训练就是让模型在预测和纠错中不断调整这些参数。同时要强调，神经网络不是人脑复制品，而是一种由许多简单计算单元连接起来的数学模型。

作业点评时，可表扬结构清楚、特征选择合理的迷你神经网络图。例如优秀作业会把“判断一部电影是否适合周末观看”画成“题材、时长、评分、同伴兴趣 → 隐藏层综合判断 → 适合/不适合”，并说明“同伴兴趣”或“时长”为什么权重更高。这类作业优秀之处在于不只是画了层次，还能解释不同输入特征怎样影响输出结果。

承接上一课的问题：神经网络需要接收输入才能计算。图片可以变成像素数字，声音可以变成波形数字，那么人类文字怎样进入模型？这节课讨论文字如何变成机器能懂的数字。

## 内容引入

**3 分钟互动：电脑真的认识“苹果”吗？**

教师在黑板写下“苹果”，让学生快速说出它表示的不同含义：水果、手机品牌、公司、颜色、味道等。随后提问：“电脑看到这两个字时，真的知道它酸甜可口，还是只看到了某种编码？”

引导学生进入核心问题：计算机不能直接理解人类文字，必须先把文字转化成数字。

## 知识点讲授

### 1. 计算机只处理数字

人类看到“苹果”会联想到水果、颜色、味道或品牌，但计算机底层处理的是数字。文字、图片、声音进入计算机后，都需要变成数字形式才能被存储和计算。

所以，AI 处理语言的第一步不是“理解”，而是把文字转换成可计算的表示。

### 2. 最简单的文字编码

一种直接做法是给每个字符一个编码。比如在 UTF-8 编码中，这几个汉字会被表示成不同的字节。UTF-8 常用十六进制表示，换成十进制后也是一组普通数字：

| 字符 | UTF-8 十六进制字节 | 对应的十进制字节 |
| ---- | ------------------ | ---------------- |
| 我   | E6 88 91           | 230 136 145      |
| 喜   | E5 96 9C           | 229 150 156      |
| 欢   | E6 AC A2           | 230 172 162      |
| 苹   | E8 8B B9           | 232 139 185      |
| 果   | E6 9E 9C           | 230 158 156      |

这些编码能让计算机区分不同的字符符号，但它们只是“代号”。例如“苹”和“果”的 UTF-8 编码放在一起，并不会告诉计算机它们组成了“苹果”这个词；“我”“喜”“欢”的编码连续出现，也不会自动告诉计算机它们组成了“我喜欢”这样的表达。

也就是说，每个字符在计算机中都有一个单纯用于区分不同字符符号的编码。这个编码没有语义关系，不会表示“苹果”是一种水果，也不会告诉计算机哪几个字应该组成一个词语。

### 3. 大模型通常先把文字切成 Token

大语言模型处理文字时，通常不会直接把整句话当成一个整体，也不一定严格按“一个汉字一个单位”或“一个词语一个单位”来处理。它会先使用分词器，把输入文字切成一个个更小的片段，这些片段常叫做 token。

例如：

```text
我喜欢人工智能。
```

在不同分词器中，可能被切成：

```text
我 / 喜欢 / 人工智能 / 。
```

也可能被切成：

```text
我 / 喜欢 / 人工 / 智能 / 。
```

英文、数字、标点、空格也可能被切成 token。token 不是固定等于“字”或“词”，而是模型实际处理文本时使用的基本片段。

分词器会把每个 token 转成一个编号，再交给模型继续计算。也就是说，大模型看到的不是人类眼中的完整句子，而是一串 token 编号。后面讲“上下文长度”和“使用成本”时，很多时候也是按 token 数量来计算的。

### 4. 词语之间其实有关系

语言中的词不是孤立的。很多词之间有相似、相反、类别、场景等关系：

- 苹果、香蕉、橘子都和“水果”有关；
- 北京、上海、广州都和“城市”有关；
- 开心、高兴、快乐意义接近；
- 开心和悲伤意义相反。

AI 要处理语言，就需要把这些关系也变成数字之间可以计算的关系。

### 5. 什么是词向量

词向量就是把一个词变成一串数字。这串数字不只是编号，还尽量包含词语的意义特征。

用多个维度描述一个学生：

```text
语文水平、数学水平、体育水平、音乐水平……
```

同样用多个维度描述一个词：

```text
是不是水果、是不是城市、情感是积极还是消极、是否与科技有关……
```

两个词在很多维度上接近时，它们的词向量也会比较接近。这样，机器就能计算“苹果”和“香蕉”比“苹果”和“书包”更相似。

**提示：** 可以把一些词投影到二维平面进行聚类，直观展示词语相似性

### 6. 为什么这很重要

大语言模型不能直接把汉字当作人类意义来处理。它要先把文字切分并转化成数字表示，再交给神经网络计算。文字数字化是语言进入 AI 系统的入口。

### 7. 知识点讲授时间安排

| 时间      | 内容                           |
| --------- | ------------------------------ |
| 3 min   | 提问：电脑懂不懂中文？         |
| 3 min   | 文字要先变成数字               |

| 3 min | 小结：语言进入 AI 前要先数字化 |

## 互动强化

### 游戏一：语义运算比拼

**5-10 分钟活动：像做数学一样做“词语关系计算”**

活动目标：让学生直观体验词向量中的一个重要想法：

> 词语之间的关系也可以被表示成方向。  
> 如果两个关系相似，就可以做出类似“北京 - 中国 + 美国 ≈ 华盛顿”的语义推理。

教师先在黑板写出示例：

```text
北京 - 中国 + 美国 = ？
```

引导学生发现：北京是中国的首都，如果把“中国”换成“美国”，答案可以是“华盛顿”或“华盛顿特区”。

教师提醒：

> 这不是严格的数学计算，而是在寻找词语之间相似的关系。有些题目可能有多个合理答案，只要关系说得通，就可以接受。

#### 1. 游戏规则

1. 教师逐题出示语义计算题。
2. 学生举手回答，并说明“为什么这样算”。
3. 如果出现多个相近答案，教师可以都写在黑板上，让全班比较哪个更贴近题目关系。
4. 每题控制在 30-60 秒，总时间约 5-10 分钟。

#### 2. 语义计算题

| 题目 | 参考答案 | 关系提示 |
| ---- | -------- | -------- |
| 北京 - 中国 + 美国 = ？ | 华盛顿 / 华盛顿特区 | 首都与国家 |
| 巴黎 - 法国 + 日本 = ？ | 东京 | 首都与国家 |
| 伦敦 - 英国 + 韩国 = ？ | 首尔 | 首都与国家 |
| 人民币 - 中国 + 美国 = ？ | 美元 | 货币与国家 |
| 日元 - 日本 + 欧元区 = ？ | 欧元 | 货币与地区 |
| 国王 - 男性 + 女性 = ？ | 女王 | 性别关系 |
| 王子 - 男性 + 女性 = ？ | 公主 | 性别关系 |
| 爸爸 - 男性 + 女性 = ？ | 妈妈 | 家庭角色 |
| 叔叔 - 男性 + 女性 = ？ | 阿姨 / 姑姑 / 婶婶 | 家庭称谓可能多解 |
| 医生 - 医院 + 学校 = ？ | 老师 / 校医 | 职业与场所，可能多解 |
| 汽车 - 公路 + 铁轨 = ？ | 火车 / 高铁 | 交通工具与行驶场景 |
| 鸟 - 天空 + 水里 = ？ | 鱼 / 鸭子 | 动物与生活环境，可能多解 |
| 钢笔 - 写字 + 画画 = ？ | 画笔 / 铅笔 | 工具与用途 |
| 冰箱 - 冷藏 + 加热 = ？ | 微波炉 / 烤箱 | 家电与功能 |
| 夏天 - 炎热 + 寒冷 = ？ | 冬天 | 季节与典型特征 |

#### 3. 教师追问

每做完几题，教师可以追问：

1. 哪些题只有一个比较明确的答案？
2. 哪些题可以有多个答案？
3. 多个答案都合理时，说明词语之间的关系是不是总能被唯一确定？
4. 如果 AI 只根据大量文本学习这些关系，它会不会也遇到模糊答案？

教师总结：

> 词向量能让机器把词语关系放进数字空间里计算，但语言本身有模糊性。同一道语义运算题可能有多个接近答案，所以 AI 的语言理解也常常是“根据上下文选择最可能的答案”。

### 游戏二：词向量训练模拟

**10-15 分钟活动：让词语在教室里“训练”出位置**

活动目标：让学生用站位变化体验词向量训练中的两个基本动作：

> 正样本：相关词靠近。  
> 负样本：不相关词远离。

#### 1. 词卡设置

只保留 4 类词，每类 3 个词，共 12 张词卡。每张词卡由 1 名学生拿在手中。

| 类别 | 词卡 |
| ---- | ---- |
| 食物类 | 苹果、香蕉、米饭 |
| 交通类 | 汽车、火车、飞机 |
| 动物类 | 猫、狗、老虎 |
| 学校类 | 老师、学生、作业 |

如果学生人数不足，可以让一名学生负责多张词卡；如果人数较多，其他学生担任“观察员”，记录哪些词逐渐靠近、哪些词逐渐远离。

**注意：** 在发放词卡的时候要打乱随机发放，不要按类别发放，让学生也随机安排站位。

#### 2. 训练句子

教师提前准备这些打乱顺序的句子，课堂中逐句读出。每句话里加粗的词都是本轮要移动的词卡词；课堂投影时也可以不加粗，让学生自己听出哪些词卡出现了。

1. 下课前，**老师**把**作业**发给**学生**，还提醒大家别忘了写名字。
2. 假期出门时，我们先坐**汽车**到车站，再坐**火车**去机场，路上，我们正好看到了**飞机**起飞。
3. 晚饭后，桌上还有一点**米饭**，妈妈又切了**苹果**和**香蕉**给大家当水果。
4. 弟弟看完**老虎**的纪录片，又翻出**猫**和**狗**的图画书，认真比较它们的样子。
5. 水果店门口摆着**苹果**和**香蕉**，隔壁小饭馆里飘来**米饭**的香味。
6. 自习课上，**学生**完成**作业**后，**老师**走过来帮大家检查错题。
7. 公园的科普展板上介绍了**猫**、**狗**和**老虎**的爪子有什么不同。
8. 早上赶路时，爸爸开**汽车**送我们到车站，旁边一列**火车**经过，远处还能看见**飞机**。
9. 科学课上，**老师**让**学生**把**作业**本打开，讨论今天观察到的现象。
10. 睡前故事里，**猫**和**狗**一起进森林，远远听见**老虎**的吼声。
11. 周末野餐时，我们把**米饭**装进饭盒，又带上**苹果**和**香蕉**当午餐。
12. 旅行回来的照片里，有公路上的**汽车**、站台旁的**火车**，还有云层下的**飞机**。

#### 3. 随机初始化

时间：1-2 分钟

让 12 名学生每人拿一张词卡，随机站在教室中。教师说明：

> 训练开始前，模型还不知道词义，所以词的位置是随机的。

#### 4. 每轮训练：正样本靠近，随机负样本远离

时间：8-10 分钟

教师依次读训练句子。每读一句，完成一轮训练：

1. 先找出句子中出现的词卡词，让它们互相靠近一步。如果一句话里有三个词卡词，可以让三个学生向中间靠拢，也可以让它们两两靠近。
2. 再请没有参与当前句移动的学生或观察员，随机说出 3 个没有出现在当前句中的词卡词，作为本轮负样本词。
3. 教师把这 3 个负样本词分别和当前句中的词卡词临时配对。每组成对词互相远离一步。

示例：

```text
晚饭后，桌上还有一点米饭，妈妈又切了苹果和香蕉给大家当水果。
```

这一轮让“米饭”“苹果”“香蕉”互相靠近一步，因为它们在相似的生活场景中一起出现。

接着，其他学生可以随机说出“飞机”“老师”“猫”这 3 个没有出现在当前句中的词。教师把它们临时配成：

```text
米饭-飞机
苹果-老师
香蕉-猫
```

这三组词互相远离一步，表示模型也在学习“哪些词没有在这个上下文中一起出现”。

再例如：

```text
假期出门时，我们先坐汽车到车站，再坐火车去机场，路上，我们正好看到了飞机起飞。
```

这一轮让“汽车”“火车”“飞机”互相靠近一步。随后其他学生可以随机说出“苹果”“作业”“狗”，教师把它们和当前句中的词组成负样本，例如“汽车-苹果”“火车-作业”“飞机-狗”，对应词互相远离一步。

经过几轮后，相关词会慢慢聚在一起，不相关词会逐渐被推开。

学生会逐渐观察到：

- 汽车、火车、飞机逐渐靠近；
- 猫、狗、老虎逐渐靠近；
- 老师、学生、作业逐渐靠近；
- 苹果、香蕉、米饭逐渐靠近。

教师说明：

> 每个句子既提供正样本，也提供负样本。句中一起出现的词会被拉近；没有在当前上下文中一起出现的随机词，会被当作负样本推远。

#### 5. 观察训练结果

时间：2-3 分钟

让学生观察现在的站位，并回答：

1. 哪些词聚在了一起？
2. 哪些词被拉远了？
3. 现在“苹果”最近的词是谁？
4. 现在“汽车”最近的词是谁？
5. 现在“老师”最近的词是谁？

预期结果：

- 苹果、香蕉、米饭形成食物区；
- 汽车、火车、飞机形成交通区；
- 猫、狗、老虎形成动物区；
- 老师、学生、作业形成学校区。

#### 6. 教师总结

教师可以用这段话收束：

> 词向量训练不是直接告诉机器“苹果是食物”。机器通过大量句子发现：苹果、香蕉、米饭经常出现在相似上下文里，所以把它们拉近。同时，每轮训练还会随机抽取没有出现在当前句中的词作为负样本，把它们和当前上下文中的词拉远。反复训练之后，词向量空间里就会自然形成语义相近的词群。



## 课后提升

**20 分钟小作业：制作 5 个词的迷你词向量**

任选 5 个词，为它们设计 4 个数字维度，例如“是否动物”“是否食物”“情感积极程度”“是否和学习有关”，每个维度 0-5 分。

完成后回答：

1. 哪两个词最相似？为什么？
2. 哪两个词差异最大？为什么？
3. 增加一个新维度后，相似度判断怎样变化？

拓展任务：查阅“word embedding”或“词嵌入”的简单介绍。
