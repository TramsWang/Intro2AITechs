# 第5课：语言模型：预测下一个词

## 前一节课内容回顾

**3 分钟回顾：从“文字数字化”到“语言生成”**

上一节课我们学习了语言进入 AI 系统前必须先变成数字。简单编号能区分不同词，但不能表达词语之间的关系；词向量用多个数字维度表示词的意义特征，让“苹果”和“香蕉”比“苹果”和“书包”在数字空间里更接近。核心结论是：大语言模型处理文字时，先处理的是可计算的数字表示。

作业点评时，可表扬维度设计有区分度、比较理由充分的迷你词向量。例如优秀作业会为“猫、狗、苹果、香蕉、学校”设计“动物程度、食物程度、学习场景、情感色彩”等维度，并能说明猫和狗为什么接近、苹果和学校为什么差异大；更好的作业还能发现增加“是否家庭宠物”这一维度后，相似度判断会发生变化。

承接上一课的问题：文字已经被转成数字，模型接下来要用这些数字做什么？对于语言模型来说，一个最核心的训练任务就是根据前面的文字预测下一个词。

## 内容引入

**3 分钟互动：你来接下一句**

教师说出几个开头，让学生快速接词：

```text
床前明月……
今天天气很好，我们一起去……
我今天早上喝了一杯……
```

学生会发现，有些开头只有一个高频答案，有些开头有多个合理答案。引导问题：大语言模型写文章，是否也是在不断预测后面最可能出现什么？

## 知识点讲授

### 1. 人类也会根据上下文预测

当我们听到“床前明月”，通常接出“光”。当我们听到“今天天气很好，我们一起去”，会接出“操场”“散步”“打球”“公园”等多个合理答案。这些答案来自我们对语言、场景和经验的判断。

预测下一个词并不只是背诵，它依赖上下文。

### 2. 语言模型做的事情

语言模型的核心任务定义为：

> 看前面的文字，猜下一个最可能出现的词。

例如：

```text
我今天早上喝了一杯……
```

模型会根据训练中见过的大量语言规律判断：

- “牛奶”的可能性较高；
- “咖啡”的可能性也较高；
- “石头”“铅笔”的可能性很低。

模型并不是只知道一个答案，而是会对很多候选词估计可能性。

### 3. 为什么预测下一个词能写文章

语言模型不是只预测一次，而是连续预测。每生成一个词，这个词又会成为新的上下文，继续影响后面的预测：

```text
人工智能 → 正在 → 改变 → 人类 → 的 → 生活
```

一次次预测连接起来，就形成句子、段落甚至文章。问答、写作、总结和翻译，都属于在特定要求下连续生成合适文本。

### 4. 它是不是在真正思考

大模型生成语言看起来像思考，但底层机制更接近：

> 根据上下文和训练中见过的大量语言规律，生成最可能合理的后续内容。

这不等于它像人一样有真实经历、情感和意识。它会写得很流畅，也会因为预测错误或事实核查不足而给出错误答案。

### 5. 概率很重要

同一个开头存在多个合理后续。例如“我喜欢吃”后面接“苹果”“面条”“火锅”都成立。语言模型会根据上下文给不同词分配不同概率。生成文本时，模型会在高概率答案中选择合适内容。

### 6. 为什么同一个问题会得到不同回答

语言模型在生成时，通常不是只面对一个候选词，而是面对一组带有不同概率的候选词。如果每一步都选择概率最高的词，回答会更稳定，但也可能比较单一；如果在高概率候选词中进行选择，回答就会有变化。

这就是为什么同一个问题问两次，大模型有时会给出不同表达，甚至选择不同角度回答。它并不是每次都从固定答案库里复制一段文字，而是在当前上下文下不断生成后续 token。

一些工具会提供生成参数，用来控制这种变化。比如：

- 温度较低：模型更倾向选择最稳妥、最高概率的表达，回答更保守；
- 温度较高：模型更容易尝试不同表达，回答更发散，但出错风险也可能增加；
- Top-p 较低：模型只在更小范围的高概率候选中选择，回答更集中；
- Top-p 较高：候选范围更大，回答可能更多样。

本课不要求记住参数公式，只要理解：大模型生成文本时既有概率规律，也有可调的随机性。

**提问：** 如何降低概率的多解性和模糊性？  
**回答：** 增加更长的上下文作为判断依据

### 7. 上下文窗口不是无限长

Transformer 能在一段文本内部比较不同 token 之间的关系，但模型一次能看到的内容并不是无限的。模型能同时处理的最大 token 数量，常被称为上下文窗口。

可以把上下文窗口理解为模型当前摊在桌面上的材料。材料在桌面上，模型就可以比较它们之间的关系；材料太多放不下，模型就可能看不到前面的一部分，或者需要先进行摘要、分段处理，再继续分析。

因此，长文本任务不只是“把所有内容都塞给模型”这么简单。上下文越长，模型能参考的信息可能越多，但计算量和成本也通常会增加。如果关键信息没有放进当前上下文窗口，模型就可能无法利用它。

### 8. 另一种训练任务：完形填空

并不是所有语言模型都只训练“预测下一个词”。以 BERT 为代表的一类模型，常见训练任务更像我们语文考试里的“完形填空”。

例如，原句是：

```text
我今天早上喝了一杯牛奶。
```

训练时，可以把其中一个词遮住：

```text
我今天早上 [MASK] 了一杯牛奶。
```

模型要根据前后文判断被遮住的词最可能是什么。这里的答案可能是“喝”，也可能是“倒”“买”等。模型同样会给不同候选词分配概率，再学习如何让正确答案的概率更高。

这种训练方式叫做**掩码语言模型**，英文常称为 Masked Language Model。它和“预测下一个词”的重要区别是：

- 预测下一个词：主要看前面的文字，适合连续生成文章；
- 完形填空：同时看前面和后面的文字，适合理解一句话或一段话的意思。

所以，像 GPT 这样的模型更擅长从左到右连续生成文本；像 BERT 这样的模型更擅长理解文本，例如判断句子情感、匹配问题和答案、给文章做分类等。

可以把两者理解为不同的训练练习：

- GPT 类模型练习“接着往下写”；
- BERT 类模型练习“根据上下文补出缺失内容”。

它们都在学习语言规律，只是练习方式和适合完成的任务不同。

## 互动强化

**20 分钟活动：串联词语编故事**

### 活动准备

教师准备 3 组相同的词卡，每组 5 张。每组安排 5 名学生参与，三组同时进行。每名学生随机拿到 1 张词卡，不能提前交换。

词卡可以选择容易组成故事但方向不同的词语，例如：

```text
雨伞、机器人、操场、秘密、明天
```

### 活动流程

1. 每组第一个拿到词卡的学生，根据自己手中的词语写一句话，作为故事开头。
2. 第二个学生拿到前一个学生写下的句子，继续写一句话。这句话必须包含自己拿到的词语，并且要和前文语义相关。
3. 第三个、第四个、第五个学生依次接力，每个人都只能看到前面已经写好的故事，并把自己的词语自然地放进续写中。
4. 最后一个学生写完后，把完整故事交给老师。
5. 三组都完成后，教师把 3 个故事读给全班听。

### 讨论引导

读完故事后，教师引导学生观察：

1. 同样是 5 个词语，不同组会写出完全不同的故事。
2. 每个学生续写时，都不是随便写一句话，而是要根据前文判断“接下来写什么才合理”。
3. 语言续写存在很大的不确定性：一个开头后面可以有很多种不同但合理的走向。
4. 上下文越长，后面的同学越容易判断故事方向，但也会受到前文限制。

总结：语言模型连续生成文本时，也是在根据已经出现的上下文进行语义续写。它每一步都有很多可能选择，最后生成的内容只是其中一种合理结果。

**20 分钟小作业：调查生活中的“语言预测”现象**

观察并查阅资料，寻找生活中使用“语言预测”技术的工具或场景，例如手机输入法联想词、搜索引擎自动补全、邮件自动补全、智能客服、翻译软件或语音助手等。

选择其中 3 个例子，完成一份简短调查报告。报告需要包含：

1. 这个工具或场景在预测什么内容？
2. 它为什么能让使用者更方便？
3. 它可能会带来哪些问题或误导？
4. 它和本节课学习的“根据上下文预测后续内容”有什么关系？

可以用表格整理：

| 场景 | 它预测什么 | 带来的方便 | 可能的问题 |
| ---- | ---------- | ---------- | ---------- |
| 手机输入法 | 下一个词或短语 | 打字更快 | 可能推荐不合适的内容 |
