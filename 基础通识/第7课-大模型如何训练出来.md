# 第7课：提升大模型的训练效率

## 前一节课内容回顾

**3 分钟回顾：从“注意力机制”到“高效训练”**

上一节课我们学习了上下文对语言理解的重要性：同一个词在不同句子中可能有不同含义，代词也需要根据前后文判断指向。注意力机制帮助模型在处理某个词时关注句子中其他相关词，Transformer 则让模型更擅长处理长文本关系，并提高训练效率。这里的“注意力”只是计算中的权重分配，不代表模型有主观意识。

作业点评时，可表扬连接准确、排序合理、依据明确的注意力连接图。例如优秀作业会从“演讲稿、图表说明、评委理解”之间找出任务目标和影响关系，能判断“它”在“请班长把它暂时放到教室后面的柜子里”中更可能指向“书包”，也能说明“透明盒子、阳光直射、温度变化、实验数据”之间的因果关系。更好的作业还会用自己的话解释 self-attention 为什么适合同时比较一句话里多个词之间的关系。

承接前两节课的问题：语言模型通过预测任务学习语言规律，Transformer 让模型更会处理上下文，也让训练可以更高效地并行进行。但大模型越大，训练需要的数据、芯片、时间和能源也越多。接下来要思考的不是“模型怎样从零训练出来”，而是“在已有训练方法和技术基础上，怎样把训练做得更快、更省、更有效”。

## 内容引入

**3 分钟互动：如果复习时间只有 2 小时**

教师提问：“如果明天就要考试，只有 2 小时复习时间，你会怎么安排？”学生可能回答：先看重点、做典型题、整理错题、请同学讲解难点、不重复做已经会的题。

继续追问：“如果只是把所有资料从头到尾机械读一遍，效率高不高？”引导学生发现：学习效果不仅取决于时间长短，也取决于资料质量、练习方法、反馈速度和资源分配。

把这个过程类比到大模型训练：提升训练效率，不是简单地少训练，而是在保证效果的前提下减少无效数据、重复计算和不必要的重新训练。

## 知识点讲授

### 1. 先区分训练和推理

讨论训练效率前，要先区分两个阶段。

训练阶段，是让模型看大量数据、完成预测任务、根据错误调整参数。这个阶段会消耗大量数据、芯片、时间和能源。

推理阶段，也就是使用阶段，是模型训练完成后，面对用户输入的问题或材料，根据已经固定下来的参数计算回答。平时我们在网页或 App 中向大模型提问，主要发生的是推理，而不是重新训练整个模型。

可以类比为：训练像长期学习，推理像考试答题。一个学生考试时会运用已经学过的知识，但不会在每道题上从小学知识重新学一遍。

### 2. 什么叫训练效率

大模型训练效率可以理解为：

> 用更少的数据、时间、算力和能源，让模型达到相近甚至更好的效果。

它不是单纯追求“训练得越快越好”。如果为了快而使用低质量数据、跳过检查或牺牲安全性，模型可能会更容易出错。真正的高效训练要同时关注：

- 训练时间是否缩短；
- 芯片和显存是否被充分利用；
- 数据是否真正有帮助；
- 模型效果是否保持稳定；
- 成本和能源消耗是否降低。

### 3. 数据效率：不是数据越多越好

第 5 课讲过，语言模型会从大量文本中练习预测后续内容。如果训练数据里有大量重复、错误、广告、低质量内容，模型会把很多时间花在学习没价值甚至有害的模式上。

提升数据效率的常见思路包括：

- 去掉重复内容，减少模型反复学习同一类材料；
- 清理明显错误、乱码、广告和无关内容；
- 选择更高质量、更有代表性的文本；
- 根据任务补充关键场景的数据；
- 保持数据多样性，避免模型只熟悉一种表达方式。

类比学习：刷 100 道重复错题，不如认真做 20 道有代表性的题并弄清楚原因。

### 4. 计算效率：让芯片少等待、多并行

训练大模型时，核心工作是对大量数字进行计算。GPU、TPU 等高性能芯片擅长同时进行大量矩阵运算，因此大模型训练非常依赖并行计算。

第 6 课讲过，Transformer 的一个重要优势是支持并行处理。训练时还可以把数据或模型分配到多块芯片上，让多个设备同时工作，再把结果汇总。这类似于把一本很厚的练习册分给多个小组同时完成，而不是一个人从第一页做到最后一页。

但并行也不是“机器越多就一定成倍变快”。设备之间需要传递信息、同步结果，如果协调不好，芯片会互相等待，效率反而下降。

### 5. 显存效率：用更聪明的方式存数字

大模型内部有大量参数，训练过程中还要保存许多中间结果。显存就像训练时的“工作台”，工作台太小，很多计算就放不下。

提升显存效率的思路包括：

- 使用更节省的数字表示，例如混合精度训练；
- 临时少保存一部分中间结果，需要时再重新计算；
- 把模型的一部分放在不同设备上协同训练；
- 让每次训练批次的大小更适合硬件能力。

可以类比为：做草稿时不是每一步都用最厚的本子完整抄一遍，而是在保证结果正确的前提下，用更省空间的记录方式。

### 6. 微调效率：不要每次都从零开始

很多应用并不需要重新训练一个完整大模型。更常见的做法是复用已经预训练好的基础模型，再针对具体任务做微调。

如果任务只是让模型适应某个领域或某种回答风格，可以使用参数高效微调方法，例如只训练一小部分新增参数，而不是修改整个模型的全部参数。这样可以显著减少训练成本和存储成本。

类比学习：一个已经会写文章的学生，要学习“写实验报告”，通常不需要从识字开始学，只需要针对实验报告的结构、语言和评分标准进行专项训练。

### 7. 基座模型、指令模型和聊天模型

同一个大模型技术路线中，模型会因为训练阶段不同而表现出不同形态。

基座模型通常指经过大量文本预训练后的模型。它已经学到许多语言规律和知识关联，但更像一个“会接着写”的模型，不一定天然懂得怎样按人类指令回答。

指令模型是在基座模型基础上进一步训练出来的。训练材料会包含“问题或任务要求 + 理想回答”，让模型更习惯按照用户指令完成总结、改写、问答、分类等任务。

聊天模型通常还会进一步适应多轮对话、人类反馈和安全要求。它不仅要回答当前问题，还要结合前几轮对话，保持语气自然，并尽量避免输出明显危险或不负责任的内容。

所以，学生要理解：平时使用的聊天大模型，不只是一个“读过很多文本”的模型，通常还经过了面向指令和对话的进一步训练。

### 8. 小模型与蒸馏：把能力变得更轻量

有些场景不一定需要最大的模型。例如校园问答、客服分类、固定格式摘要等任务，可以训练更小的模型来完成。

模型蒸馏可以理解为：让一个能力强的大模型当“老师”，把它的解题思路和回答方式教给一个更小的“学生模型”。小模型能力可能不如大模型全面，但运行和后续训练成本更低，在明确任务中可能已经够用。

### 9. 效率提升需要评估

提高训练效率不能只看速度，还要检查模型是否真的变好了。常见评估包括：

- 回答是否准确；
- 是否能遵守指令；
- 是否在不同题型上稳定；
- 是否减少了错误和偏见；
- 是否在成本降低后仍然保持可用效果。

总结：高效训练不是偷工减料，而是让数据、算法、硬件和工程方法配合得更好。

## 互动强化

**20 分钟活动：有限预算训练挑战**

### 教学目的

通过模拟“预算有限”的训练决策，让学生理解训练效率来自多方面选择：数据质量、并行计算、显存管理、微调方法和效果评估都会影响最终结果。

### 活动设定

全班分成若干小组。每组要设计一个“校园问答助手”的训练方案，目标是让它能回答课程安排、图书馆规则、社团活动等问题。

每组有 10 个预算点，只能从下面策略中选择若干项：

| 策略 | 预算 | 作用 |
| ---- | ---- | ---- |
| 收集大量未经筛选网页 | 3 | 数据多，但噪声和错误也多 |
| 收集高质量校内资料 | 3 | 数据更可靠，和任务更相关 |
| 数据去重和清洗 | 2 | 减少重复、错误和无关内容 |
| 补充特殊场景样本 | 2 | 覆盖节假日、转专业、借书异常等少见问题 |
| 从零训练一个模型 | 6 | 成本高，适合非常通用的大任务 |
| 在已有模型上微调 | 3 | 复用基础能力，适合具体任务 |
| 参数高效微调 | 2 | 只调整少量参数，训练更省 |
| 使用多块芯片并行训练 | 3 | 训练更快，但需要协调 |
| 使用混合精度训练 | 1 | 减少显存占用，提高计算效率 |
| 训练后做效果评估 | 2 | 检查回答是否准确、稳定、安全 |
| 蒸馏成小模型 | 2 | 后续使用更省，但能力范围可能变窄 |

### 活动流程

1. 每组在 10 个预算点内选择训练策略。
2. 每组写出选择理由，说明哪些策略能提升效率，哪些策略可能带来风险。
3. 每组必须回答三个问题：
   - 你们怎样减少无效数据？
   - 你们怎样避免从零训练带来的浪费？
   - 你们怎样判断模型训练后真的可用？
4. 每组选一名代表用 1 分钟介绍方案。
5. 全班讨论：哪组方案最省？哪组方案最稳？哪组方案风险最大？

### 课堂引导

教师引导学生发现：

1. 只买“大量未经筛选网页”并不一定高效，低质量数据会浪费训练。
2. “已有模型 + 高质量资料 + 数据清洗 + 效果评估”通常比盲目从零训练更适合具体应用。
3. 并行训练、混合精度和参数高效微调都能提升效率，但都需要在效果评估中验证。
4. 不同目标对应不同方案：通用大模型追求广泛能力，校园问答助手更重视相关数据和稳定回答。

## 课后提升

**20 分钟小作业：设计一个高效训练方案**

请选择一个具体应用场景，例如校园问答助手、作文批改助手、古诗学习助手、社团活动推荐助手等，设计一份简短的高效训练方案。

方案需要包含：

1. 这个模型要完成什么任务？
2. 需要准备哪些高质量数据？
3. 哪些数据应该清洗或去掉？
4. 为什么不一定需要从零训练？
5. 可以使用哪一种提高训练效率的方法？
6. 训练后应该用什么问题或标准来评估效果？

可以用表格整理：

| 训练环节 | 我的选择 | 为什么更高效 | 可能风险 |
| -------- | -------- | ------------ | -------- |
| 数据准备 | 收集校规、课程表、常见问答 | 数据和任务直接相关 | 资料过期会导致回答错误 |

拓展任务：查阅“LoRA”“混合精度训练”或“模型蒸馏”中的一个概念，用自己的话说明它为什么能节省训练成本。
