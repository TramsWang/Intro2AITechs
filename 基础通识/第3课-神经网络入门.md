# 第3课：神经网络入门

## 前一节课内容回顾

**3 分钟回顾：从“数据中找规律”到“更复杂的模型”**

上一节课我们学习了机器学习的基本思想：传统程序依靠人提前写好明确规则，而机器学习是让机器通过大量样本和标签总结规律，再用训练得到的模型预测新样本。几个关键词需要学生重新说出含义：样本是给机器看的例子，标签是正确答案，训练是不断调整判断方式，模型是训练后用于预测的工具。

教师公布课后拓展参考答案：上一节课中的水果分类、猫狗识别都属于有监督学习，因为训练数据里不仅有样本，还有对应的正确标签，例如“这张图是猫”“这个水果是香蕉”。机器通过样本和标签之间的对应关系，学习以后怎样给新样本预测答案。无监督学习则不提前给标准答案，机器面对的是一批没有标签的数据，它要自己寻找相似性、分组或隐藏规律。可以用一句话概括区别：有监督学习像“带答案做例题”，无监督学习像“没有答案，先自己观察这些数据可以怎样分组”。

作业点评时，可表扬目标清楚、样本和标签对应准确的分类任务。例如优秀作业会先说明“要让机器判断垃圾属于哪一类”，再列出纸巾、塑料瓶、果皮、电池等样本及标签，并写出机器可能依据“材质、是否可回收、是否有害”等特征学习规律。这类作业优秀之处在于把“想让机器做什么”和“机器凭什么判断”都写清楚了。

承接上一课的问题：如果任务只是水果分类，几个简单特征也许够用；但图片识别、语音识别、翻译和写作会同时涉及大量线索。机器怎样综合这么多线索做判断？这节课进入神经网络。

## 内容引入

**3 分钟互动：一句话为什么不只看关键词？**

教师展示一段话，先不公布标准判断，让学生快速说出这段话的感情基调是“开心”还是“难过”：

```text
今天考试成绩出来了，我比上次进步了一点，老师也说我最近很努力。放学后朋友们拉着我去吃甜品，还一直讲笑话逗我开心，我也跟着笑了很久。可是回家的路上，我想起自己离目标分数还差很多，又担心爸妈会失望。手机里弹出朋友发来的合照，照片里的我笑得很灿烂，但我盯着看了半天，心里还是有点空落落的。
```

学生给出判断后，教师追问依据：有人会关注“进步了一点”“老师表扬”“吃甜品”“讲笑话”“笑了很久”等偏开心的线索，有人会关注“离目标分数还差很多”“担心爸妈会失望”“心里还是有点空落落的”等偏难过的线索，也有人会注意表面行为和真实感受并不完全一致。不同学生会关注不同线索，因此可能给出不同判断。

引导学生思考：判断一段话的情感基调时，人类不会只看一个关键词，而是会综合词语、语气、转折和上下文。机器也需要一种能综合复杂线索的模型。

## 知识点讲授

### 1. 为什么需要神经网络

简单问题用简单规则解决，例如分数是否及格、年龄是否达到标准、温度是否超过某个数值。但复杂问题很难靠几条规则解决：

- 判断一段话的情感基调是开心还是难过；
- 翻译一段文字；
- 写一篇结构完整的文章；
- 从语音中识别出文字。

本节课用同一个例子贯穿讲解：让机器判断一段话的情感基调是“开心”还是“难过”。

例如：

```text
今天我收到了朋友准备的生日惊喜，大家一起唱歌、吃蛋糕，我觉得特别温暖。
```

大多数人会判断这段话偏“开心”。但如果只写一条简单规则，例如“出现生日就是开心”，就很容易出错：

```text
今天是我的生日，可是没有人记得，我一个人坐了很久。
```

这段话也出现了“生日”，但整体更接近“难过”。所以复杂判断不能只看一个词，而要综合很多线索：哪些词出现了、这些词怎样组合、有没有转折、整段话的氛围是什么。神经网络适合处理这类需要综合多种线索的问题。

### 2. 神经网络像什么

神经网络可以类比为一个“班级投票系统”。假设全班一起判断下面这段话是开心还是难过：

```text
虽然今天下雨了，但我和朋友聊天聊到很晚，心情一下子变好了。
```

不同同学可以分别关注不同线索：

- 有同学看到“下雨”，觉得可能偏难过；
- 有同学看到“朋友”“聊天”，觉得可能偏开心；
- 有同学注意到“虽然……但……”，发现后半句才是重点；
- 有同学看到“心情一下子变好了”，判断开心的信号很强。

最后把许多判断综合起来，就会得到更可靠的结果。神经网络中的每个节点也类似：每个节点只做一点简单计算，许多节点连接在一起，就能完成复杂判断。

**引申：** 神经网络不是死记硬背某个关键词，而是在学习数据中更复杂的分布和组合关系。（线性 -> 非线性）

### 3. 输入层、隐藏层、输出层

神经网络分成三类部分。仍然以“判断一段话开心还是难过”为例：

```text
输入层：接收这段话中的信息
隐藏层：加工和组合这些信息
输出层：给出“开心”或“难过”的判断
```

可以把流程写成：

```text
一段话 → 神经网络处理 → 输出“开心”或“难过”
```

输入层接收文字信息。为了让机器处理文字，通常要先把文字转成数字，例如把词语、词的位置、词与词之间的关系变成模型能理解的表示。

隐藏层负责逐步加工这些信息：

- 第一层可能注意到一些局部线索，例如“惊喜”“温暖”“一个人”“失望”；
- 后面的层会组合更复杂的线索，例如“虽然前面不好，但后面变好了”；
- 更深的层会形成整体判断：这段话主要是在表达开心，还是主要在表达难过。

输出层给出最后结果。它可以不是简单说一个答案，而是给出两个分数：

```text
开心：0.86
难过：0.14
```

分数更高的一类就是模型的预测结果。

**案例：** 参考CNN隐藏层识别人脸特征的直观案例

### 4. 参数和训练

课堂中可以把参数解释为“每条信息的重要程度”。在判断情感时，有些线索贡献更大，模型就会给它们更高权重。

例如：

```text
今天是我的生日，可是没有人记得。
```

如果模型只把“生日”看得很重要，就可能错误判断为“开心”。训练的作用就是让模型通过大量样本慢慢调整参数：

1. 给模型看很多带标签的例子，例如“这段话是开心”“这段话是难过”；
2. 模型先根据当前参数做预测；
3. 如果预测错了，就调整参数；
4. 反复多次后，模型会逐渐学到哪些线索更重要、哪些组合更关键。

经过训练后，模型不仅会关注“生日”这类单个词，也会学会重视“可是没有人记得”这样的上下文。也就是说，训练让神经网络从大量例子中学会更合理的判断方式。

### 5. 神经网络和人脑有什么相似

“神经网络”这个名字借鉴了人脑神经元的启发。课堂中可以先用人脑做类比：人脑里有很多神经元，每个神经元会接收一些信号，再把处理后的信号传给其他神经元。大量神经元连接在一起，人就能综合许多信息做判断。

神经网络也有类似的思路：每个节点接收一些数字信息，进行简单计算，再把结果传给下一层。很多节点连接起来，就能把零散线索组合成整体判断。

在“判断开心还是难过”的例子里，可以这样类比：

- 人看到“笑了很久”“吃甜品”，会感到这段话有开心的线索；
- 人看到“担心爸妈会失望”“心里空落落”，又会感到这段话有难过的线索；
- 人脑会把这些线索综合起来，形成一个整体印象；
- 神经网络则把这些文字线索转成数字，通过很多节点和参数计算，最后输出“开心”或“难过”的概率。

但也要提醒学生：神经网络不是人脑的复制品。它不会真正感受到快乐或伤心，它做的是数学计算。这个类比的作用是帮助理解“许多简单节点连接起来，可以共同完成复杂判断”。

## 互动强化

**20 分钟活动：校园植物分类家**

让学生组成一个“神经网络”，根据校园植物的观察特征，判断它更可能属于哪一类植物。

### 📌 大类与四个小类（不变）

- 阔叶乔木
- 针叶/常绿乔木
- 开花灌木
- 草本花卉

### 🧩 第一层：直接特征打分（不变）

| 编号 | 特征 | 说明 |
|------|------|------|
| A | 木质化茎干 | 1=草质茎，10=坚硬木质 |
| B | 叶片形状 | 1=针形/鳞形，10=扁平宽阔 |
| C | 植株高度 | 1=低于30cm，10=高于5米 |
| D | 开花明显度 | 1=几乎无花或不显眼，10=花大色艳 |

### 🔁 第二层：组合特征打分（全部重新设计）

第二层产生 **4 个组合特征**，每个都是 A、B、C、D 的函数，不再直接等于任何原始特征。

| 组合特征 | 计算公式 | 含义解释 |
|----------|----------|----------|
| E（木本高大指数） | `0.5×A + 0.5×C` | 木质化程度与高度的综合 → 乔木倾向 |
| F（叶形阔度指数） | `0.7×B + 0.3×(1 - |A-C|/10)` | 不仅看叶形，还考虑木质与高度是否一致（若A和C都高，则加分，因为大树通常阔叶） |
| G（矮小柔嫩指数） | `0.6×(10 - C) + 0.4×(10 - A)` | 低矮且非木质 → 灌木/草本倾向 |
| H（花显眼综合值） | `0.6×D + 0.4×(1 - A/10)` | 开花明显度结合非木质（草本/灌木更容易花显眼，木质大树花往往不显眼） |

> 说明：公式中的 `|A-C|/10` 表示木质与高度的差异程度，差异越小（A和C都高或都低）则该项得分高，乘以0.3后加入F，使F更综合。H中加入 `(1 - A/10)` 使得草质茎（A低）时H更高。

### 🎯 第三层：最终类别得分（再组合）

四个类别的得分由第二层的 **E, F, G, H** 加权求和，不再直接使用 A,B,C,D。

| 类别 | 计算公式 | 设计逻辑 |
|------|----------|----------|
| 阔叶乔木 | `0.5×E + 0.4×F + 0.1×(10 - G)` | 高木本高大 + 阔叶特征强 + 不矮小 |
| 针叶/常绿乔木 | `0.6×E + 0.4×(10 - F)` | 高木本高大 + 非阔叶（F低则10-F高） |
| 开花灌木 | `0.5×G + 0.4×H + 0.1×E` | 矮小柔嫩 + 花显眼 + 少量木本特征 |
| 草本花卉 | `0.6×G + 0.4×H` | 极矮小柔嫩 + 花显眼 |

### 📊 计算示例

#### 示例1：樟树（阔叶乔木）

- 玩家打分：A=9, B=9, C=8, D=3
- 第二层计算：
  - E = 0.5×9 + 0.5×8 = 8.5
  - |A-C| = |9-8| = 1, 1 - 1/10 = 0.9，F = 0.7×9 + 0.3×0.9 = 6.3 + 0.27 = 6.57
  - G = 0.6×(10-8) + 0.4×(10-9) = 0.6×2 + 0.4×1 = 1.2 + 0.4 = 1.6
  - H = 0.6×3 + 0.4×(1 - 9/10) = 1.8 + 0.4×0.1 = 1.8 + 0.04 = 1.84
- 第三层得分：
  - 阔叶乔木 = 0.5×8.5 + 0.4×6.57 + 0.1×(10-1.6) = 4.25 + 2.628 + 0.84 = 7.718
  - 针叶乔木 = 0.6×8.5 + 0.4×(10-6.57) = 5.1 + 0.4×3.43 = 5.1 + 1.372 = 6.472
  - 开花灌木 = 0.5×1.6 + 0.4×1.84 + 0.1×8.5 = 0.8 + 0.736 + 0.85 = 2.386
  - 草本花卉 = 0.6×1.6 + 0.4×1.84 = 0.96 + 0.736 = 1.696
- **结果：阔叶乔木** ✅

#### 示例2：月季（开花灌木）

- 打分：A=6（半木质），B=6，C=3，D=9
- 第二层：
  - E = 0.5×6 + 0.5×3 = 4.5
  - |A-C| = 3，1 - 3/10 = 0.7，F = 0.7×6 + 0.3×0.7 = 4.2 + 0.21 = 4.41
  - G = 0.6×(10-3) + 0.4×(10-6) = 0.6×7 + 0.4×4 = 4.2 + 1.6 = 5.8
  - H = 0.6×9 + 0.4×(1 - 6/10) = 5.4 + 0.4×0.4 = 5.4 + 0.16 = 5.56
- 第三层：
  - 阔叶乔木 = 0.5×4.5 + 0.4×4.41 + 0.1×(10-5.8) = 2.25 + 1.764 + 0.42 = 4.434
  - 针叶乔木 = 0.6×4.5 + 0.4×(10-4.41) = 2.7 + 0.4×5.59 = 2.7 + 2.236 = 4.936
  - 开花灌木 = 0.5×5.8 + 0.4×5.56 + 0.1×4.5 = 2.9 + 2.224 + 0.45 = 5.574
  - 草本花卉 = 0.6×5.8 + 0.4×5.56 = 3.48 + 2.224 = 5.704
- 此时草本花卉得分略高，说明公式可能需要微调。教师可调整第三层权重，例如将开花灌木公式改为 `0.5×G + 0.5×H`（去掉E）或增加G的权重。这本身就是学习过程：学生可以讨论为什么月季容易被判定为草本花卉，从而理解灌木与草本的区分难点。

### 注意：可加入迭代修正参数环节

上述公式中的权重参数是比较理想化的结果。课堂互动刚开始时，教师可以故意保留一组“不够好”的参数，让网络先犯错，再带领学生通过几轮迭代修正参数。

#### 1. 固定一组训练样本

先准备 4 张校园植物样本卡，每张卡写出真实类别和第一层打分。

| 样本 | 真实类别 | 建议打分 |
|------|----------|----------|
| 樟树 | 阔叶乔木 | A=9, B=9, C=8, D=3 |
| 雪松 | 针叶/常绿乔木 | A=9, B=2, C=9, D=1 |
| 月季 | 开花灌木 | A=6, B=6, C=3, D=9 |
| 菊花 | 草本花卉 | A=2, B=7, C=2, D=9 |

#### 2. 第 0 轮：用初始参数预测

第二层公式先保持不变，只让学生调整第三层参数。先使用上面的初始第三层公式进行预测，并把每个样本的预测结果记录下来。

以月季为例，初始结果是：

```text
开花灌木 = 5.574
草本花卉 = 5.704
```

网络把月季误判为“草本花卉”。这时不要直接告诉学生“正确公式”，而是追问：

> 月季和草本花卉都可能低矮、开花明显，所以 G 和 H 都高。
> 但月季有一定木质化，草本花卉通常更不木质化。
> 哪个组合特征可以帮助区分它们？

学生应能发现：E（木本高大指数）虽然不高，但可以作为区分灌木和草本的辅助线索。

#### 3. 第 1 轮：根据错误微调参数

设定一条简单调参规则：

```text
如果正确类别分数偏低：提高正确类别对关键特征的权重。
如果错误类别分数偏高：降低错误类别对混淆特征的权重，或加入一个负向权重。
每轮只改 1-2 个参数，每次改 0.1，避免一下子改成“标准答案”。
```

针对月季被误判为草本花卉，可以先保留“开花灌木”公式不变，只给“草本花卉”增加一个对 E 的负向权重：

```text
调整前：草本花卉 = 0.6×G + 0.4×H
调整后：草本花卉 = 0.6×G + 0.4×H - 0.1×E
```

重新计算月季：

```text
草本花卉 = 0.6×5.8 + 0.4×5.56 - 0.1×4.5
        = 3.48 + 2.224 - 0.45
        = 5.254
```

此时：

```text
开花灌木 = 5.574
草本花卉 = 5.254
```

月季就会被判为“开花灌木”。这说明一次小幅调参已经让模型纠正了一个错误。

#### 4. 第 2 轮：检查有没有“修坏”其他样本

调参后要重新测试菊花，确认草本花卉不会被误伤。

菊花的第二层关键值大致为：

```text
E = 2
G = 8
H = 5.72
```

调整后：

```text
开花灌木 = 0.5×8 + 0.4×5.72 + 0.1×2 = 6.488
草本花卉 = 0.6×8 + 0.4×5.72 - 0.1×2 = 6.888
```

菊花仍然会被判为“草本花卉”，说明这次调整没有破坏原本正确的样本。

#### 5. 第 3 轮：全样本复测并继续迭代

最后让学生把樟树、雪松、月季、菊花全部重新算一遍，记录每一轮的结果。

| 轮次 | 发现的问题 | 参数调整 | 复测结果 |
|------|------------|----------|----------|
| 第 0 轮 | 月季被误判为草本花卉 | 暂不调整 | 发现错误 |
| 第 1 轮 | 草本花卉对半木质植物过于宽松 | 草本花卉加入 `-0.1×E` | 月季改判为开花灌木 |
| 第 2 轮 | 检查是否影响真正的草本 | 用菊花复测 | 菊花仍为草本花卉 |
| 第 3 轮 | 全部样本复查 | 若还有错误，再按同样规则微调 | 得到一版更合理的参数 |

这个过程要让学生看到：训练不是一次性写出完美公式，而是根据错误样本不断调整参数，使正确类别得分更高、错误类别得分更低。
