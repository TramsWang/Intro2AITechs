# 课程设计

## 1. 课程总路线设计

### 1.1 课程总目标

通过 4 节智能体基础课，让学生在已经理解大模型基础能力和局限的前提下，进一步理解：

1. 大模型智能体不是一个“更会聊天的大模型”，而是围绕目标持续完成任务的系统；
2. 可以用 PaE 框架理解智能体：感知任务与环境、分析和规划下一步、执行动作并观察反馈；
3. 智能体的各个环节本质上都离不开与大模型的精心编排互动，提示词是组织这些互动的重要接口；
4. 好的提示词需要明确任务、背景、约束和输出形式，并遵循清楚、具体、分步、可检查的原则；
5. 不同大模型适合不同场景，模型选型要综合任务难度、上下文长度、多模态能力、工具调用能力、成本、速度和安全要求；
6. ReAct、多智能体等框架能帮助大模型把“思考、行动、观察”组织成可追踪的工作流；
7. CoT、ToT、GoT 等思维模式可以引导模型进行分步推理、多路线比较和图状关系分析，但输出仍需检查；
8. 工具调用、MCP 和 RAG 让智能体连接计算器、搜索、数据库、文件、代码运行器和知识库等外部能力；
9. 记忆、评估和安全边界决定了智能体能否被负责任地用于真实学习和工作场景；
10. 后续实践课中使用智能体时，学生需要学会把需求写清楚、把工具权限给清楚、把输出检查清楚，而不是把责任完全交给 AI。

### 1.2 与基础通识课的衔接

基础通识课已经帮助学生理解了：

- 大模型通过语言建模生成文本；
- Transformer 和注意力机制帮助模型处理上下文；
- 大模型可能产生幻觉，也需要事实核查和隐私保护；
- 清楚提问、提供背景、分步检查，比直接让模型“一口气完成所有事情”更可靠。

智能体基础课在这些内容之上继续推进：

```text
大模型能回答问题
  ↓
通过提示词组织任务、背景、约束和输出
  ↓
用 PaE / ReAct 等框架把任务变成循环工作流
  ↓
调用工具和外部知识补足模型能力
  ↓
通过记忆、评估和安全边界形成可检查的智能体系统
```

因此，本模块的重点不是让学生立刻编写复杂智能体代码，而是先建立实践前必须具备的概念框架：智能体由什么组成，提示词怎样驱动智能体流程，如何选择大模型，怎样规划任务，怎样调用工具和外部知识，哪些事情不能完全自动化。

### 1.3 课程案例

本模块继续使用“学习支持智能体”作为全局案例，但不再使用学习日记文本作为驱动材料，而是改为具体学习使用场景。主场景设定为：**考前知识点复习智能体**。

设定为：学校希望设计一个受老师监管的 AI 学习支持智能体，帮助学生在考试前整理复习目标、定位薄弱知识点、生成复习计划和自测题。它不能替代老师做最终评价，也不能保证成绩提升，但可以完成一些学习辅助任务：

1. 接收学生的具体学习请求，例如考试时间、科目、考试范围、薄弱知识点、可用复习时间；
2. 把“帮我复习”拆成知识点梳理、薄弱点诊断、复习顺序安排、自测题生成、错因归纳和计划调整；
3. 调用不同工具，例如课程知识库、错题记录、日程表、题目生成器、答案解析器和安全规则检查器；
4. 基于外部知识库检索相关概念、例题和教师提供的复习材料；
5. 生成给学生使用的复习卡片、每日计划、自测题和错题回顾建议；
6. 在不确定或高风险场景下提示学生向老师确认，例如考试范围不明确、题目答案存在争议、学习压力过高或请求涉及作弊。

贯穿课堂的示例请求可以使用：

```text
下周三要考物理，我对电路分析和电磁感应总是混淆。每天晚上大概有 45 分钟复习时间。请帮我整理重点、安排 5 天复习计划，并给我几道自测题。
```

在 4 节课中，同一场景承担不同教学任务：

- 第1课：分析这个请求怎样被 PaE 框架理解，怎样写成更清楚的提示词，怎样选择合适的大模型；
- 第2课：把复习目标放入 PAE、ReAct 和多智能体框架，比较 CoT、ToT、GoT 对任务拆解和规划的影响；
- 第3课：说明智能体怎样调用课程知识库、日程工具、题目工具和 RAG 检索；
- 第4课：用 OpenClaw 案例反观学习支持智能体的能力边界、记忆风险、评估方法和安全责任。

### 1.5 单节课时间节奏

每节课建议采用相对稳定的节奏：

1. 前课回顾与问题导入：约 3 到 5 分钟；
2. 知识讲解：约 20 分钟，集中讲清本节 3 到 5 个核心概念；
3. 演示或互动强化：约 15 到 20 分钟，通过提示词改写、框架图、工具调用模拟、案例审查会等方式让学生把概念用起来；
4. 课后提升：约 20 分钟的小任务，可作为作业或下一节课开头的讨论材料。

## 2. 每节课简要设计

详细课纲已拆分为单独文件。本节只保留整体设计层面的课程定位、关键内容和活动形式。

### 2.1 第1课：大模型智能体的定义与提示词工程

本课建立智能体的基本概念：智能体不是“更会聊天的大模型”，而是围绕目标持续完成任务的系统。学生通过“考前物理复习”案例理解 PaE 框架，知道感知、分析、执行和反馈分别对应哪些动作。

关键内容包括：普通聊天和智能体任务的区别、PAE 框架、提示词四要素、提示词四原则、常见提示词类型和大模型选型原则。

课堂活动以“把模糊请求改成智能体提示词”为主，让学生把“帮我复习物理，我快考试了”改写成包含角色、任务、背景、输出、约束和检查点的结构化提示词，并进行模型选型判断。

详细课纲见：[第1课-大模型智能体的定义与提示词工程.md](第1课-大模型智能体的定义与提示词工程.md)。

### 2.2 第2课：大模型智能体的框架

本课让学生理解智能体不是只有一种实现方式，而是可以通过不同框架组织任务流程。学生在回顾 PaE 的基础上学习 ReAct 的“判断依据、行动、观察”循环，并理解多智能体的角色分工。

关键内容包括：PAE 与 ReAct 的关系、多智能体架构、任务拆解与规划、CoT、ToT、GoT，以及任务拆解提示、角色分工提示、示例约束提示、多方案比较提示和自我检查提示。

课堂活动以“同一任务，不同框架”为主。学生围绕同一个复习请求分别画出 PaE 流程、写出 ReAct 循环、设计多智能体分工，并判断哪些任务适合 CoT、ToT 或 GoT。

详细课纲见：[第2课-大模型智能体的框架.md](第2课-大模型智能体的框架.md)。

### 2.3 第3课：工具调用与外部知识

本课让学生理解大模型本身主要擅长语言理解和生成，智能体要完成真实任务，常常需要调用外部工具和知识库。学生需要理解工具调用不是模型“自己打开工具”，而是由模型生成结构化请求，外部程序执行后返回结果。

关键内容包括：工具调用原理、工具调用流程、工具名称、参数、返回值、权限、MCP 的基本作用、常见工具类型、RAG 流程、向量检索与 Word Embedding 的联系，以及工具和 RAG 的风险。

课堂活动以“人体工具调用模拟”为主。学生分别扮演用户、大模型、工具管理员、课程知识库、日程工具、题库工具、安全审核员和记录员，模拟从用户请求到工具调用、返回结果、生成计划和安全审核的全过程。

详细课纲见：[第3课-工具调用与外部知识.md](第3课-工具调用与外部知识.md)。

### 2.4 第4课：案例剖析与智能体的利弊

本课通过 OpenClaw 案例理解真实智能体系统的能力和风险。学生把前面学过的 PAE、ReAct、工具调用、记忆和安全边界放到一个更完整的系统中审查，理解智能体越能做事，越需要权限控制、日志、评估和人工确认。

关键内容包括：用 PaE 和 ReAct 复盘案例、智能体平台的优势、权限过大和工具连接带来的风险、上下文与长期记忆、日志、评估维度、安全边界，以及学习支持智能体中“可以自动做、必须确认后做、完全不应该做”的分类。

课堂活动以“OpenClaw 案例审查会”为主。学生从目标、数据、记忆、工具、权限、评估、人工审核和输出风险八个角度审查一个学校学习支持智能体草案，并改造成更负责任的版本。

详细课纲见：[第4课-案例剖析与智能体的利弊.md](第4课-案例剖析与智能体的利弊.md)。

## 3. 整体教学风格设计

这套智能体基础课面向已经完成大模型基础学习的高中生，采用“概念承接 + 提示词实操 + 框架建模 + 工具体验 + 案例审查”的方式推进。每节课建议控制为约 20 分钟知识讲解和 15 到 20 分钟互动。

### 3.1 四段式课程结构

每节课通常拆成“前课回顾、内容引入、知识点讲授、互动强化、课后提升”几个部分：

1. 前课回顾：用 2 到 3 分钟承接基础通识课或上一节智能体课的核心概念；
2. 内容引入：用学生熟悉的考前复习、错题整理、资料查找和学习计划场景提出问题；
3. 知识点讲授：围绕少量关键词展开，例如 PAE、提示词、模型选型、ReAct、多智能体、工具调用、MCP、RAG、记忆、评估和边界；
4. 互动强化：通过提示词改写、框架图绘制、人体工具调用、RAG 排序、案例审查会等方式，让学生亲自体验智能体工作流；
5. 课后提升：要求学生把概念迁移到自己的学习场景，为后续实践项目积累想法。

### 3.2 三少三多

#### 3.2.1 少拟人化，多系统化

不把智能体讲成“AI 有了自己的想法”，而是讲成一个系统：

- 大模型负责理解、生成、规划和解释；
- 提示词负责组织任务、背景、约束和输出；
- 框架负责安排感知、分析、行动、观察和反馈；
- 工具负责外部计算、检索、读写和执行；
- 记忆负责保存必要状态；
- 评估负责检查结果质量；
- 人类负责确认高风险操作和承担最终判断。

#### 3.2.2 少自动化崇拜，多边界意识

智能体看起来更像“会做事”，但课堂要反复强调：

- 能自动执行，不代表应该自动执行；
- 能生成建议，不代表建议一定正确；
- 能保存记忆，不代表可以保存所有信息；
- 能调用工具，不代表可以拥有无限权限；
- 自托管和本地运行不等于天然安全；
- 涉及隐私、安全、心理健康、考试公平、金钱和重要决定时，必须有人类确认。

#### 3.2.3 少抽象术语，多可视化流程

对高中生讲智能体，需要把术语落到可画、可演示、可检查的流程：

```text
目标
  ↓
提示词组织
  ↓
框架规划
  ↓
选模型
  ↓
选工具和知识库
  ↓
执行与观察
  ↓
评估与调整
  ↓
记录、记忆与审核
```

每个技术概念都尽量落到一个可操作问题：

- PAE：系统现在感知什么、分析什么、执行什么；
- 提示词：怎样把任务说清楚；
- 模型选型：这个任务需要什么模型能力；
- ReAct：下一步推理、行动和观察是什么；
- 多智能体：哪些角色需要分工；
- 工具调用：用哪个工具、传什么参数；
- MCP：工具和数据源怎样统一接入；
- RAG：答案依据来自哪些外部材料；
- 记忆：哪些信息可以保存，保存多久；
- 评估：结果是否完成任务；
- 安全：哪些地方必须停下来问人。

## 4. 最终学生应达到的理解水平

学完后，学生不需要会开发完整智能体框架，但应该能用自己的话解释：

> 大模型智能体是在大语言模型基础上构建的一类系统。它不只是回答问题，而是通过提示词和框架组织感知、分析、执行和反馈，围绕目标进行任务拆解，选择合适模型和工具，检索外部知识，观察结果，并根据反馈调整下一步。PAE、ReAct、多智能体、CoT、ToT、GoT、工具调用、MCP 和 RAG 都是帮助我们理解或实现智能体工作流的方法。智能体可以提升学习支持效率，但它仍然会犯错，可能选错工具、误解目标、保存不该保存的信息，或把不可靠结果说得很确定。因此，设计和使用智能体时必须明确目标、写清提示词、限制权限、保护隐私、记录关键步骤、设置人工确认，并对结果进行评估。

这 4 节课要建立的核心认知是：

```text
大模型生成能力
  ↓
提示词工程
  ↓
智能体框架
  ↓
任务拆解与思维模式
  ↓
工具调用与外部知识
  ↓
记忆、评估与安全边界
```

学生完成本模块后，应能为后续实践课做好准备：知道自己要让智能体完成什么学习任务，知道任务需要怎样的提示词、模型、工具、知识库和步骤，也知道哪些结果必须检查、哪些权限必须谨慎、哪些场景必须交给人来判断。
