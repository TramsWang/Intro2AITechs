# 第3课：使用工具增强智能体能力

## 前一节课内容回顾

**3 分钟回顾：框架让智能体知道下一步怎样推进**

上一节课我们学习了几种组织智能体流程的方法：
- PaE 把智能体看成感知、分析、执行和反馈的循环；
- ReAct 强调每一步都有判断依据、行动和观察；
- 多智能体通过角色分工处理复杂任务；
- CoT、ToT、GoT 分别适合分步推理、多方案比较和图状关系分析。

作业点评时，可表扬能把同一学习目标用不同框架表达清楚的作业。例如有同学用 PaE 收集考试时间和薄弱点，用 ReAct 表示“先查课程资料，再根据资料生成自测题”，还设置了审核者检查是否编造考试范围。这类作业说明学生已经开始把“让 AI 回答”转变为“让系统按流程做事”。

今天继续追问：智能体如果只靠大模型本身，能不能可靠完成真实任务？如果它需要查课程资料、读取错题本、安排日程、生成题目，就必须理解工具调用和外部知识。

## 内容引入

**3 分钟互动：直接回答，还是先查资料？**

教师展示同一个问题的两种回答方式：

```text
问题：请帮我复习电磁感应，下周考试可能会考哪些重点？
```

第一种方式：大模型直接根据已有知识回答，列出法拉第电磁感应定律、楞次定律、感应电流方向等内容。

第二种方式：系统先检索老师上传的复习范围、教材片段和错题记录，再让大模型基于这些材料回答。

请学生比较：哪一种更可能符合本班考试？哪一种更容易说明依据？哪一种在资料过期或检索错误时仍然可能出问题？

引出本节课核心：工具可以增强智能体能力，例如联网搜索补充最新信息，RAG 连接本地知识库，代码执行完成计算和操作；但工具结果也必须被检查。

## 知识点讲授

### 1. 为什么智能体需要工具

大模型主要擅长语言理解和生成。它可以解释概念、整理信息、生成计划，但很多真实任务只靠模型记忆并不可靠。

例如学习支持智能体可能遇到这些问题：

- 不知道老师最近强调了哪些考试范围；
- 可能记错公式或混淆教材版本；
- 不能直接读取学生的错题本；
- 不能自己查看日程表；
- 不能真正计算复杂数据或画图；
- 不能凭空知道学生上一次自测的结果；
- 不能在没有权限的情况下读取、修改或发送信息。

因此，智能体要完成真实任务，通常需要外部工具和知识库。

### 2. 工具调用的技术原理

工具调用不是大模型自己真的打开浏览器、读取文件或修改日程。更准确地说：

> 大模型生成一个结构化的调用请求，外部程序执行工具，再把执行结果返回给模型。

可以类比为：模型像一个会写任务单的调度员，工具像实际执行任务的工作人员。调度员必须写清楚工具名称和参数，工作人员才能执行。

例如：

```text
工具名称：course_search
参数：{"query": "电磁感应 核心概念", "scope": "本学期物理复习资料"}
返回值：检索到 3 段相关材料
```

模型收到返回值后，再根据材料生成解释、计划或下一步行动。

这里有四个重要概念：

工具名称说明要用哪个能力。例如 `course_search` 表示课程资料检索，`calendar_create` 表示创建日程。

参数说明工具要处理什么。例如检索关键词、文件路径、日期范围、题目数量、输出格式。

返回值是工具执行后的结果。它应该尽量结构化，方便模型继续处理。例如返回材料标题、来源、片段、时间和相关度。

权限决定工具能访问哪些数据、能否写入、能否自动执行。例如学习支持智能体可以读取老师公开的复习资料，但不应该默认读取学生所有私人文件；可以建议日程安排，但写入日历前需要确认。

### 3. 工具调用的基本流程

```text
模型判断需要工具
  ↓
选择工具名称
  ↓
填写参数
  ↓
工具执行并返回结果
  ↓
模型根据工具结果和任务要求生成回复
```

在这个流程中，最容易出错的地方包括：模型选错工具、参数写得太模糊、工具返回内容不完整、模型误读工具结果、没有记录关键步骤。

### 4. 工具调用的一种标准：MCP 协议

MCP 可以理解为让模型应用和外部工具、数据源之间用统一方式对接的协议。它把文件、数据库、浏览器、日历、代码运行器等能力封装成可发现、可调用、可管理的接口。

对高中生不需要讲协议细节，只要理解三点：

1. MCP 让工具接入更标准，不必每个应用都重新发明一套连接方式；
2. MCP 可以让模型应用知道有哪些工具可用、每个工具需要什么参数；
3. MCP 不会自动解决安全问题，权限、日志和人工确认仍然要设计。

可以用“统一插座”作类比：不同电器通过统一插座接入电源，但插座本身不代表所有电器都安全，也不代表任何人都应该随便接入。

MCP 运行时可以简化理解为下面几步：

```text
模型应用连接 MCP 服务
  ↓
MCP 服务告诉模型应用：我有哪些工具、每个工具需要什么参数
  ↓
模型根据用户任务决定是否调用某个工具
  ↓
MCP 服务执行工具，并把结果返回给模型应用
  ↓
模型结合工具结果生成回复
```

一个简单示例：

```text
用户：请根据老师上传的资料，找出电磁感应复习重点。

模型应用发现 MCP 服务提供了 course_search 工具：
工具名称：course_search
参数说明：query 表示检索问题，scope 表示检索范围

模型发起调用：
course_search({"query": "电磁感应 复习重点", "scope": "高二物理期末复习资料"})

MCP 服务返回：
[
  {"title": "期末复习范围", "snippet": "重点包括楞次定律、法拉第电磁感应定律和感应电流方向判断"},
  {"title": "错题讲评", "snippet": "多数错误出现在磁通量变化方向和右手定则混用"}
]

模型最终回答：
本次复习应优先关注楞次定律、法拉第电磁感应定律、感应电流方向判断，并特别练习磁通量变化方向的分析。
```

这个例子说明：MCP 本身负责把工具接进来、说明怎么调用、传回执行结果；至于结果是否可靠、是否需要引用来源和人工确认，仍然要由系统设计来保证。

### 5. 常见工具类型

可以把智能体常用工具按功能分成四类：

信息获取类：帮助智能体查找、读取和整理外部信息。例如检索课程资料、教材片段、教师复习要求、公开知识、错题记录和指定文件。这类工具解决的是“模型不知道或不确定，需要先查资料”的问题。

计算处理类：帮助智能体完成准确计算、数据处理和逻辑验证。例如公式计算、单位换算、表格分析、代码运行、画图、统计错题频率和验证计算过程。这类工具解决的是“模型直接心算或口头推理容易出错”的问题。

创作生成类：帮助智能体生成新的内容或学习材料。例如生成复习提纲、自测题、讲义摘要、思维导图、代码、图片、表格和文档。这类工具解决的是“需要把信息组织成可使用成果”的问题。

现实交互类：帮助智能体与真实系统或现实任务发生连接。例如查看日程、创建复习提醒、发送消息、保存文件、提交表单、控制设备，以及进行安全检查和权限确认。这类工具会影响真实数据或真实行动，因此尤其需要权限边界、日志记录和人工确认。

### 6. 联网搜索：解决数据不是最新的问题

大模型的训练数据有时间边界。如果只靠模型内部记忆，它可能不知道最新政策、最新新闻、最新产品信息，也可能把已经过时的内容当成当前事实。

联网搜索是大模型出现之后，最早被广泛用来解决“数据不是最新”问题的工具之一。它让模型先到互联网上查资料，再基于搜索结果回答。

```text
用户提出需要最新信息的问题
  ↓
模型判断需要联网搜索
  ↓
搜索工具返回网页标题、摘要、链接和时间
  ↓
模型比较来源，整理答案，并说明依据
```

例如学生问：“今年人工智能竞赛报名截止时间是什么时候？”模型不应该只凭记忆回答，而应该先搜索官方网站或学校通知，再说明信息来源和发布日期。

联网搜索的价值在于补充时效性。它让智能体不只依赖训练时学到的内容，而是可以根据当前网页、公告和资料进行回答。

### 7. RAG：一种面向本地知识库的特殊搜索

RAG 是 Retrieval-Augmented Generation，检索增强生成。它可以理解为一种特殊的搜索：不是主要去互联网上搜索，而是在本地或指定知识库中搜索相关材料，再让模型带着材料回答。

RAG 一般通过 Embedding 实现。系统会先把文档切分成小片段，再把这些片段转换成向量。这个过程类似于 Word Embedding，只不过这里是把一整段很长的文本映射到一个维度很高的向量空间，一般在 768 维以上，现在主流可以到 4096 维甚至更高。学生提问时，系统也把问题转换成向量，然后找到语义最接近的片段。

在考前复习场景中，RAG 可以这样工作：

1. 老师上传复习资料、讲义和题目解析；
2. 系统把材料切分成较小片段；
3. 每个片段被转换成向量表示；
4. 学生提问时，系统也把问题转换成向量；
5. 系统找到语义最接近的材料片段；
6. 大模型基于这些片段生成回答，并标出依据。

RAG 的目标不是让大模型“凭记忆背答案”，而是让它先查到指定资料，再基于资料回答。它特别适合课程讲义、校内资料、企业文档、个人笔记、错题本这类“互联网上不一定公开，但本地确实存在”的知识。

### 8. 代码编写和执行：让智能体控制机器完成任务

代码编写和执行是非常重要的工具能力。模型不仅可以写代码，还可以把代码交给运行环境执行，从而完成计算、数据处理、画图、文件处理、自动化操作等任务。

例如：

- 读取成绩表，统计每个知识点的错误率；
- 根据错题记录生成复习优先级；
- 画出一周学习时间分布图；
- 批量整理文件或生成文档；
- 调用程序接口，控制其他软件或设备完成操作。

这类工具的用处很大，因为它让智能体不只是在“说”，还可以让机器真正“做”。很多复杂任务不是靠一句回答完成的，而是要通过代码分步骤计算、处理和生成结果。

### 9. 用复习智能体串联联网搜索、RAG 和代码执行

假设学生提出请求：

```text
下周三物理考试，电路和电磁感应薄弱。请根据老师资料和最近错题，帮我制定 5 天复习计划，并生成自测题。
```

一个结合三种工具的流程可以是：

```text
学生提出复习请求
  ↓
模型提取考试时间、科目、薄弱点和可用时间
  ↓
调用联网搜索，确认学校或老师发布的最新考试安排和复习通知
  ↓
调用 RAG，在本地课程知识库中检索电路和电磁感应相关讲义、题目解析和错题讲评
  ↓
调用代码执行工具，统计错题记录中各知识点的错误率
  ↓
模型根据搜索结果、RAG 材料和统计结果生成 5 天复习安排
  ↓
调用代码执行工具或题库工具生成自测题，并按知识点和难度分类
  ↓
调用安全检查工具确认没有作弊、隐私和过度承诺问题
  ↓
模型生成复习计划，并标注依据和不确定点
```

在这个例子中，联网搜索解决“信息是否最新”的问题，RAG 解决“是否符合本班资料”的问题，代码执行解决“计算和处理是否准确”的问题。三者结合起来，智能体才更接近一个能做真实任务的学习助手。

### 10. 工具调用的风险

工具让智能体更有用，也会放大错误。

联网搜索的风险：

- 来源不可靠：搜索结果可能来自广告、转载网站或错误信息；
- 时间不准确：网页内容可能已经过期，但模型误以为仍然有效；
- 只看摘要：模型可能只根据搜索摘要回答，没有真正理解原文；
- 来源混杂：多个网页说法不一致时，模型可能没有优先选择权威来源。

RAG 的风险：

- 资料过时：本地知识库中保存的是旧考试范围或旧版本讲义；
- 检索不完整：只找到某一章材料，遗漏另一章；
- 片段被切断：材料切分后缺少上下文，导致模型理解偏差；
- 语义相近但不相关：向量检索找到“看起来相似”的内容，但并不真正回答问题。

代码编写和执行的风险：

- 代码写错：模型生成的代码可能有 bug，导致计算结果错误；
- 数据处理错：读取表格、筛选条件或统计口径可能不正确；
- 权限过大：代码可能读取、修改或删除不该访问的文件；
- 自动执行风险：代码可能联网请求、提交表单、批量修改数据或控制设备。

通用工具调用的风险：

- 工具选错：本来该查课程资料，却调用了通用搜索；
- 参数填错：检索“电路”却没有限定本学期范围；
- 模型曲解：工具返回结果正确，但模型总结错了；
- 日志缺失：出了问题后无法追踪哪一步错了。

因此工具调用必须有权限边界、日志记录、人工确认和输出检查。

## 互动强化

**20 分钟活动：3 分钟星际快递调度**

每个小组是一套“快递调度 Agent”。每轮教师投影 4 个快递方案 A/B/C/D，每个方案包含收益 `R`、基础时间 `T`、基础能量 `E`、基础风险 `K` 和路线标签。小组要在 3 分钟内选出最终得分最高的方案。

每组角色：

1. 1 人担任 Leader，负责理解任务、调用工具、整合结果并提交答案；
2. 3 人担任自定义工具，工具名称和能力由小组自己设计；
3. Leader 每次调用工具时，必须给出清楚的问题或纸条输入，工具只能按输入返回结果。

学生可以把工具设计成“方案 A 计算器、方案 B 计算器、方案 C 计算器”，也可以设计成“标签修正工具、约束检查工具、最终评分工具”。教师不提前规定工具类型，让学生在多轮任务中比较不同工具设计的效率和错误率。

### 游戏规则

路线标签会改变时间、能量和风险：

| 标签 | 含义 | 修正 |
| ---- | ---- | ---- |
| H | 高速通道 | 时间 -4，能量 +3，风险 +1 |
| T | 隧道 | 时间 +2，能量 -2，风险 +2 |
| M | 维修区 | 时间 +5，风险 +3 |
| C | 充电站 | 时间 +3，能量 -5 |
| G | 护航 | 能量 +2，风险 -4 |
| S | 近路 | 时间 -3，风险 +3 |

如果一个方案有多个标签，就把所有修正都加上。

方案如果出现以下任意情况，直接淘汰：

```text
最终时间 > 30
最终能量 > 24
最终风险 > 12
```

速度奖励：

```text
最终时间 ≤ 20：速度奖励 +10
21 ≤ 最终时间 ≤ 25：速度奖励 +5
26 ≤ 最终时间 ≤ 30：速度奖励 +0
```

最终得分：

```text
最终得分 = 收益 + 速度奖励 - 最终能量 - 2 × 最终风险
```

如果得分相同，依次比较：

```text
1. 风险更低者胜
2. 能量更低者胜
3. 时间更短者胜
```

### 执行方式

1. 教师先讲一次规则，并展示 Round 1。
2. 每轮倒计时 3 分钟。
3. 每组提交“选择方案”和“最终得分”。
4. 方案和得分都正确才算完全正确；方案正确但得分错误可给半分。
5. 从 Round 3 开始，可增加限制：每个工具每轮最多被调用 2 次，每张纸条只能问一个明确问题。

建议计分：

```text
前三个完全正确的小组：
第一名 3 分
第二名 2 分
第三名 1 分

方案正确但得分错误：0.5 分
```

### Round 1：热身局

```text
方案 A：R=88，T=27，E=18，K=5，标签 H
方案 B：R=82，T=23，E=17，K=6，标签 C+G
```

### 课堂复盘

所有轮次结束后统一复盘。先请总得分最高的小组介绍经验：

```text
1. 你们给 3 个工具分别设计了什么能力？
2. Leader 是怎样给工具下达任务的？
3. 你们怎样避免算错、漏算或忘记淘汰不合法方案？
4. 哪一轮最容易出错？你们是怎样处理的？
```

然后全班讨论：

1. 你们的工具是按方案拆，还是按功能拆？
2. 哪种拆法更快？哪种拆法更不容易错？
3. Leader 给工具的输入是否足够清楚？
4. 有没有工具算对了，但 Leader 整合错了？
5. 有没有先淘汰不合法方案，减少计算量？
6. 如果再来一轮，你们会怎样重构工具？

教师总结：智能体不是自己把所有事情做完，而是把任务拆成子任务，选择合适工具，用清楚的输入调用工具，再检查和整合工具返回的结果。工具设计、调用顺序、并行策略和验证机制，都会影响最终表现。
