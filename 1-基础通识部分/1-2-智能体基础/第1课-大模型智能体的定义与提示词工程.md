# 第1课：大模型智能体的定义与提示词工程

## 前一模块内容回顾

**3 分钟回顾：从“大模型会回答”到“智能体会围绕目标做事”**

在大模型基础课中，学生已经知道：
- 大模型通过学习大量语言材料，学会根据上下文预测和生成文本；
- Transformer 和注意力机制帮助模型处理较长上下文；
- 大模型可以问答、总结、写作和对话，但也可能产生幻觉；
- 清楚提问、提供背景、分步检查，比一句话让模型完成复杂任务更可靠。

本节课承接这些认识，继续追问：如果一个 AI 不只是回答一句问题，而是能围绕目标拆解任务、调用工具并输出有结构的结果，它还是普通聊天机器人吗？

教师可以提醒学生：智能体不是“有了自主意识的大模型”，而是在大模型基础上，把提示词、流程、工具和检查机制组织起来的系统。

## 内容引入

**3 分钟互动：同一句“帮我复习物理”，会发生什么？**

教师在屏幕或黑板上写出学生常见请求：

```text
帮我复习物理，我快考试了。
```

请学生判断：如果把这句话发给普通聊天机器人，它可能会直接给出一段通用复习建议，例如“先看课本、再做题、最后复盘错题”。这样的回答有帮助，但它不知道考试时间、考试范围、学生薄弱点和每天可用时间。

再展示一条更完整的请求：

```text
下周三要考物理，我对电路分析和电磁感应总是混淆。每天晚上大概有 45 分钟复习时间。请帮我整理重点、安排 5 天复习计划，并给我几道自测题。
```

引导学生发现：第二条请求已经包含目标、背景、约束和输出要求。一个学习支持智能体不应该只给一段建议，而应该先理解任务、拆解步骤、判断需要哪些资料和工具，再生成计划和自测题，最终输出可执行结果。

本节课的核心问题是：什么是大模型智能体？怎样用提示词把任务说清楚，让大模型更稳定地参与智能体流程？

## 知识点讲授

### 1. 什么是大模型智能体

大模型智能体可以理解为：

> 在大语言模型基础上构建的、能够围绕目标持续完成任务的系统。

它通常会接收用户目标，理解当前环境，拆解任务，选择工具或下一步行动，并按计划输出结果。

这里要避免一个误解：智能体不是大模型突然“有了自己的想法”。更准确地说，它是一个由人设计的系统。系统把大模型放在关键位置，让大模型负责理解语言、生成计划、解释结果和判断下一步；同时通过提示词、工具、权限、日志和人工确认来约束它。

### 2. 普通聊天和智能体任务的区别

用考前复习场景对比：

| 普通聊天 | 智能体任务 |
| -------- | ---------- |
| 用户问一句，模型答一句 | 围绕目标持续推进 |
| 容易给出通用建议 | 基于用户定义的具体需求和场景给出针对性回复 |
| 主要靠模型已有知识回答 | 可以调用资料、日程、题库等工具 |
| 输出后通常结束 | 会根据反馈调整下一步 |
| 结果较难追踪 | 可以记录依据、工具调用和检查点 |

例如“帮我复习物理”如果作为普通聊天，模型可能直接输出一份复习建议。如果作为智能体任务，系统更应该完成这些动作：

1. 追问缺失信息，例如考试范围、年级、教材版本、剩余天数；
2. 把任务拆成知识点梳理、薄弱点诊断、复习计划、自测和复盘；
3. 检索课程资料或错题记录；
4. 生成可执行的每日安排；
5. 根据自测结果调整后续计划；
6. 对不确定内容标注“需要向老师确认”。

### 3. 智能体的简单思路：先规划，再执行

智能体其实就是在模拟人类解决问题时的通用的、一般的过程。先规划再执行就是最简单、最基础的方案。遇到问题时，人类会先将复杂任务拆解成多个前后关联的子任务，形成一份解决方案，其中的每个子任务都比原任务简单，执行起来比较直接。接下来，再按照这个方案具体执行每一个子任务：

```text
Plan 计划
  ↓
Execute 执行并输出结果
```

在学习支持智能体中，智能体需要先在 Plan 阶段理解用户目标、考试时间、科目范围、薄弱知识点、可用复习时间、已有错题和课程资料，再把任务拆成清晰步骤，确定先做什么、后做什么，以及最终要输出什么结果。然后，在 Execute 阶段中，智能体根据 Plan 阶段得到的计划完成具体任务，例如生成知识卡片、调用知识库、安排日程、生成自测题、批改答案、整理错因，并直接输出执行结果。

### 4. PaE 两个环节都离不开提示词

智能体不是只在开始时写一句模糊请求。Plan 和 Execute 两个环节都需要我们明确告知大模型应该怎么做。

Plan 环节可以这样要求大模型：

```text
请从学生请求中提取考试时间、科目、薄弱知识点、可用复习时间和期望输出。
请根据学生目标，把任务拆成 4-6 个可执行步骤。
每个步骤说明目的、需要的信息、可能使用的工具和最终输出要求。
如果信息缺失，请列出需要追问的问题。
```

Execute 则这样要求大模型：

```text
用户下周三要考物理，他对电路分析和电磁感应总是混淆。每天晚上大概有 45 分钟复习时间。请帮他整理重点、安排 5 天复习计划，并设计几道自测题。

按照如下思路完成这个任务，给出每一步的执行结果：
1. 第一步：……
2. 第二步：……
……
```

### 5. 提示词是什么

我们刚才用于引导大模型的任务说明就是大家常说的“提示词”。它通常包含任务、要求、格式和场景，是控制模型行为的重要接口之一。

提示词不只是“问一句话”。在智能体系统中，提示词常常承担这些作用：

- 说明要完成什么任务；
- 说明完成任务时需要遵守的要求；
- 规定输出格式；
- 提供任务发生的具体场景；
- 要求模型列出依据、不确定点和下一步。

### 6. 提示词四要素

一个适合智能体任务的提示词通常包含四个要素。

| 要素 | 说明 | 举例 |
| ---- | ---- | ---- |
| 任务 | 说明要完成什么 | 请帮助学生整理电路分析和电磁感应的核心知识点，并安排 5 天复习计划。 |
| 要求 | 说明完成任务时必须遵守的条件 | 每天复习时间不超过 45 分钟；不要编造考试范围；无法确认的信息必须标注“需要向老师确认”。 |
| 格式 | 规定结果的呈现形式 | 请用表格输出每日复习计划，并附 6 道自测题。 |
| 场景 | 提供任务发生的具体背景 | 学生下周三考试，每天晚上约 45 分钟复习，电路分析和电磁感应容易混淆。 |

把四个要素合在一起，就比“帮我复习物理”稳定得多。

### 7. 提示词四原则

| 原则 | 说明 | 正面例子 | 反面例子 |
| ---- | ---- | -------- | -------- |
| 全面性 | 覆盖完成任务所需的关键信息，包括目标、对象、范围、要求、格式和场景，避免遗漏影响结果的重要条件。 | 学生下周三考物理，每天 45 分钟，重点复习电路分析和电磁感应，请输出 5 天计划和 6 道自测题。 | 帮我复习物理。 |
| 一致性 | 任务、要求、格式和场景之间不能互相冲突。 | 请安排 5 天复习计划，并用 5 行表格分别列出每天内容。 | 请安排 5 天复习计划，用 3 行表格输出，每行 1 天。 |
| 简洁性 | 表达清楚但不堆砌无关信息，保留会影响输出的条件，删去与任务无关的背景和重复要求。 | 请围绕电路分析和电磁感应安排 5 天复习，每天不超过 45 分钟。 | 我太难了，物理老师讲课很快，我总感觉听不懂，请帮我复习一下，拜托拜托！ |
| 合理性 | 要求要符合真实场景和可执行条件。 | 高一物理中的电磁感应部分要重点复习。 | 高一物理中的《出师表》课文要重点复习。 |

### 8. 大模型选用原则

不同大模型适合不同任务。选模型时不应该只问“哪个最强”，而要看任务需要什么能力。

- 简单整理、摘要、格式转换：可以使用速度快、成本低的模型；
- 复杂推理、复习规划、工具调用：优先使用推理能力强、工具调用稳定的模型；
- 长资料复习：选择上下文窗口长，或者能和检索系统配合的模型；
- 图片、公式、手写题目：选择多模态能力强的模型；
- 隐私敏感或校内资料：优先考虑本地模型、私有部署或严格权限控制；
- 高频练习和批量生成：综合考虑成本、速度和质量评估；
- 涉及安全、学业评价和学生隐私：需要更严格的人工确认和日志记录。

结合当前大模型的实际产品发布情况给出例子。比如 Qwen 系列模型，GPT 的不同型号。

## 互动强化

**20 分钟活动：把模糊请求改成智能体提示词**

1. 教师给出原始请求：“帮我复习物理，我快考试了。”
2. 每组用 4 分钟把它改写成包含任务、要求、格式、场景的结构化提示词。
3. 每组必须加入至少 2 条约束，例如“不确定考试范围要追问”“不要保证成绩提升”“计划不能超过每天 45 分钟”。
4. 学生把提示词输入大模型，观察输出是否包含计划、自测题、追问或风险提示。
5. 每组把检查要求补充进提示词，例如要求计划必须控制在每天 45 分钟以内，并覆盖电路和电磁感应。
6. 教师抽取 2-3 组展示，对比模糊提示词和结构化提示词的输出差异。
7. 最后进行模型选型快问快答：教师给出“总结一页课本”“批改手写卷照片”“处理含学生成绩的数据”等任务，学生判断需要普通模型、多模态模型、强推理模型、本地模型还是带工具调用的模型。

## 课后提升

**20 分钟小作业：写一份自己的学习智能体提示词**

选择一个真实学习任务，例如复习某一章、整理错题、准备演讲、制定阅读计划或练习英语写作。写出一版完整提示词，必须包含：

1. 任务；
2. 要求，至少包含 2 条约束和 1 个检查要求；
3. 格式；
4. 场景；
5. 你认为应该选择哪类大模型，并说明理由。

拓展任务：用同一个任务分别测试“模糊提示词”和“结构化提示词”，比较两次输出在准确性、可执行性和可检查性上的差别。
